IMPORTANT: Pour être en mesure de voir les notifications, il faut que le plugin soit activé (voir Onglet configuration) mais aussi que vous ayez coché la case Activer les notifications dans vos préférences utilisateur

Introduction

Le plugin notification permet d'afficher sur l'interface d'administration des notifications concernant n'importe quelle fonction de Dotclear (billets/pages/commentaires/plugins/thèmes/etc...)

Onglet Composants enregistrés

Liste des composants
Notifications fonctionne grâce à des composants. Ces derniers sont un moyen de regrouper visuellement le type de notifications qui est affichée.
Chaque ligne représente un composant avec:
  • son nom
  • le lien vers la définition des permissions relatives au composant. Ces permissions permettent de définir quels utilisateurs peuvent voir les notifications
  • l'image du composant: elle sera affichée dans le coin supérieur gauche de chaque notification appartenant à ce composant
  • son statut:
    • actif: toutes notifications relatives à ce composant seront enregistrées et affichées
    • non actif: toutes notifications relatives à ce composant ne seront pas enregistrées et ne seront pas affichées
Actions sur les composants
Chaque composant peut être activé/désactivé à volonté. Pour effectuer ces opérations, il suffit de sélectionner les composants voulus, sélectionner l'action à faire dans le selecteur en bas à droite de la liste et cliquer sur ok

Définition des permissions

Les types de notifications
A chaque notification, relative à un composant, est affecté un type:
  • new: pour toute action créant un nouveau contenu
  • upd: pour toute action mettant à jour du contenu
  • del: pour toute action supprimant du contenu
  • msg: pour toute action nécessitant un message d'information
  • err: pour toute action nécessitant un message d'erreur
  • spm: pour toute action considérée comme spam
Choix des permissions
Pour chacun de ces types, il est possible de personaliser l'affichage des notifications suivant les autorisations des utilisateurs. Les permissions affichées dans le selecteur sont les mêmes que sur page de définition de permission d'un utilisateur sur chaque blog.
Par exemple, pour le composant Post, type new, si vous définissez la permission à administrateur, seuls les utilisateurs ayant des droits supérieurs ou égaux à cette permission veront ce type de notifications

Onglet Configuration

Activer les notifications
Activation générale du plugin notification. Permet aux utilisateurs de choisir ou non d'afficher les notifications relatives à leurs permissions
Coller les notifications
Une notification collée signifie qu'elle reste affichée tant que l'utilisateur n'aura pas cliquer sur le bouton de fermeture de celle ci ou le bouton Fermer tout
Afficher toutes les notifications de tous les blogs
Affiche sur le blog courant les notifications relatives au blog ainsi qu'aux autres blogs de la plateforme. Cet affichage n'est visible que pour les administrateurs et super administrateurs
Position des notifications
Défini la position de l'affichage des notifications
Temps d'affichage des notifications
Temps pour lequel chaque notification reste affichée. Ce temps doit être défini en secondes. Cette option n'est prise compte que si la case Coller les notifications n'est pas cochée
Temps entre chaque requête
Intervale de temps entre deux requêtes pour la vérification de nouvelles notifications. Ce temps doit être défini en secondes
Nettoyer automatiquement les notifications
Defini si le plugin doit vider automatiquement la table des notifications pour celles consommées
