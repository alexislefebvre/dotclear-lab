IMPORTANT: To be able to see notifications, plugin has to be activated (see Configuration tab) and you have also to check the checkbox Enable notifications in user preferences

Introduction

Notifications plugin allow to display in admin interface notifications about any Dotclear functions (entries/pages/comments/plugins/themes/etc...)

Registered components tab

Components list
Notifications works by components. It's a way to group each kind of notifications
Each line represent a component with:
  • his mane
  • the link to the his permissions definition. Those permissions define who can see these notifications
  • Component icon: will be display on a top left corner of each notification
  • his status:
    • enable: Each notifications related to this component are saved and displayed
    • disable: All notifications related to this component are not saved and displayed
Components actions
Each component can be enabld/disabled. To perfom those operations, select components and the action to do on the combobox at the right bottom of the list and press ok

Permissions definition

Notifications types
For each notification, related to a component, a type is defined:
  • new: for all actions creating content
  • upd: for all actions updating content
  • del: for all actions deleting content
  • msg: for all actions needing information message
  • err: for all actions needing error message
  • spm: for all actions considering as spam
Permission choice
For each of those types, it's possible to custom notifications display regarding users permissions. Displayed permissions in the combobox are the same than one on user permissions page.
For example, for component Post, type new, if you define the permission to administrator, only users who have permissions equal of greater to this permission will see this type of notifications

Configuration tab

Enable notifications
General plugin activation. Allow users to display or not notifications related to their permissions
Sticky notifications
A sticky notification means it stays displayed until user close it
Display all notifications of all blogs
Display on the current blog all notifications related to all blogs of the farm. This display is visible only for administrators and super administrators
Position of notifications
Defines position of notifications display
Time to display notifications
Time for notification display. This time should be in second. This option is used only if checkbox Sticky notifications is not checked
Time beetween each request
Time interval beetween two requests of new notifications check. This time should be in second.
Auto clean notifications
Defines if notifications plugin should clean automatically notifications table of consummed notifications
