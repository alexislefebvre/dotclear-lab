A propos

Cette extension permet d'ajouter simplement une licence à tous le fichier d'une extension.
Les changements se font sur les modules installés. Ces changements sont irréversibles.

Paramètres des fichiers

Ecraser la licence existante
Réécrit la licence dans les fichiers même si elle existe déjà.
Ajouter la licence aux entêtes de fichiers
Permet de selectionner les types de fichiers auquels il faut ajouter la licence.
Exclusion
Permet d'exclure certains fichiers ou dossiers. Dans ce champs il faut utiliser une expression rationnelle.
Par exemple pour ne pas affecter les fichiers de traduction, il faut écrire:
/(\/locales\/)/

Paramètres des astuces

Ces paramètres utilisent les behaviors de certains plugins pour automatiser les ajouts de licence.

Ajouter la licence avant de créer un paquetage avec le plugin pacKman
Si vous avez le plugin pacKman d'installé et que cette option est activé, la licence sera ajoutée au module avec la création du paquetage.
Ajouter la licence après avoir créé une traduction avec le plugin translater
Si vous avez le plugin translater d'installé et que cette option est activé, la licence sera ajoutée aux fichiers de traduction lors de la création de ces derniers.
Attention la liste d'exclusion est prioritaire.

Paramètres des entêtes

Les entêtes sont les résumés de licence. Elles sont placées en haut de chaque fichier d'un module sous forme de commentaire. Pour les plugins il est préfèrable de choisir la licence gpl2 qui correspond à la licence de Dotclear. Il est possible de modifier une entête et certaines informations directement en rapport avec le module peuvent être ecrites dans une entête en utilisant les masques suivant:

%year%
Année en cours.
%module_id%
Id du module. (Nom correspondant au dossier du module)
%module_name%
Le nom du module tel qu'il est définie dans le fichier _define.php.
%module_author%
Le nom de l'auteur du module tel qu'il est définie dans le fichier _define.php.
%module_type%
Soit plugin, soit theme.
%user_cn%
Le nom commun de l'utilisateur qui utilise licenseBootstrap.
%user_name%
Le nom de l'utilisateur qui utilise licenseBootstrap.
%user_email%
L'adresse email de l'utilisateur qui utilise licenseBootstrap.

Un fichier LICENSE est également écrit à la racine du module avec le texte complet de la licence choisie.

Remise à zero

Un bouton de remise à zero est disponible. Il permet de remettre les options et les entêtes de licenseBootstrap à l'origine.
