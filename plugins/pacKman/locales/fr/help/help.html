À propos

Cette extension permet de créer des paquetages au format zip de plugins et thèmes pour Dotclear 2.

Paramètres

Activer le menu sur la page des extensions
Ajoute un onglet de création de paquetage sur la page de gestion des extensions.
Écraser les paquetages existants
Écrit les fichiers de paquetage même si un fichier de même nom existe dèjà (le plus récent écrase le précédent).
Nom du paquetage exporté
C'est le nom du paquetage qui sera créé. Il ne faut pas ajouter l'extension zip à la fin car elle est automatiquement ajoutée. Ce nom peut être précédé d'un nom repertoire mais ce dernier doit exister dans le dépôt. Les balises suivantes seront automiquement remplacées par les propriétés correspondantes du module :
  • %type% : plugin ou thème,
  • %id% : id du module,
  • %version% : version du module,
  • %author% : auteur du module,
  • %time% : le timestamp actuel.
Par exemple:
%type%-%id%
Nom du second paquetage exporté
Un deuxième paquetage peut être créé en même temps. Cela peut être pratique pour sauvegarder chaque version.
Ce champ suit les même règles que le précédent.
Par exemple:
old/%type%-%id%-%version%
Chemin vers le dépôt
C'est le chemin complet (depuis la racine) du dossier ou seront créées les sauvegardes. Il est préférable et même utile d'utiliser le dossier public.
Ce dossier doit être accessible en écriture.
Fichiers supplémentaires à exclure du paquetage
Simple liste de noms des fichiers à exclure. Le joker * peut être utilisé dans ces noms. Les noms sont séparées par des points-virgules.
Par exemple:
*.zip;*.tar;*.tar.gz
Par défaut les fichiers suivants sont exclus:
.;..;__MACOSX;.svn;CVS;.DS_Store;Thumbs.db.
