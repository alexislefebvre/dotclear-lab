À propos

Cette extension permet de créer des paquetages au format zip de plugins et thèmes directement depuis l'interface d'administration de Dotclear.

Racine

Chemin vers le dépôt
Le chemin sur le serveur ou seront stockés les paquetages générés par ce plugin. Il est recommandé d'utiliser sous répertoire d'un dossier public d'un blog. Ce répertoire doit être accessible en lecture/écriture pour le plugin.

Fichiers

Nom du paquetage exporté
C'est le nom du paquetage qui sera créé. Il ne faut pas ajouter l'extension zip à la fin car elle est automatiquement ajoutée. Ce nom peut être précédé d'un nom de repertoire mais ce dernier doit exister dans le dépôt. Les balises suivantes seront automiquement remplacées par les propriétés correspondantes du module :
  • %type% : plugin ou thème,
  • %id% : id du module,
  • %version% : version du module,
  • %author% : auteur du module,
  • %time% : le timestamp actuel.
Par exemple:
%type%-%id%
Nom du second paquetage exporté
Un deuxième paquetage peut être créé en même temps. Cela peut être pratique pour sauvegarder chaque version.
Ce champ suit les mêmes règles que le précédent.
Par exemple:
old/%type%-%id%-%version%
Écraser les paquetages existants
Si un paquetage de même nom existe déjà dans le répertoire, le paquetage sera alors remplacé par celui fraichement créé.

Contenu

Fichiers supplémentaires à exclure du paquetage
Simple liste de noms des fichiers à exclure. Le joker * peut être utilisé dans ces noms. Les noms sont séparées par des points-virgules.
Par exemple:
*.zip;*.tar;*.tar.gz
Par défaut les fichiers suivants sont toujours exclus:
.;..;__MACOSX;.svn;.hg*;.git*;CVS;.DS_Store;Thumbs.db
Retirer les commentaires des fichiers
Lors de la création du paquetage les commentaires des fichiers PHP sont retirés à la volée, cela n'affecte pas les fichiers sources.
