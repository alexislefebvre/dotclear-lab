A propos

Cette extension permet de créer des paquetage au format zip de plugins et thèmes pour Dotclear 2.

Paramètres

Activer le menu sur la page des extensions
Ajoute un onglet de création de paquetage sur la page de gestion des extensions.
Écraser les paquetages existants
Ecrit les fichiers de paquetage même si un fichier de même nom existe dèjà.
Nom du paquetage exporté
C'est le nom du paquetage qui sera crée. Il ne faut pas ajouter l'extension zip à la fin car elle est automatiquement ajouté. Ce nom peut être précèdé d'un nom repertoire mais ce dernier doit éxisté dans le dépôt. Les balise suivante seront automiquement remplacé par les propriétés correspondantes du module:
  • %type% : plugin ou thème,
  • %id% : id du module,
  • %version% : version du module,
  • %author% : auteur du module,
  • %time% : le timestamp actuel,
Par exemple:
%type%-%id%
Nom du second paquetage exporté
Un deuxième paquetage peut être crée en même temps. Cela peut-être pratique pour sauvergarder chaque version.
Ce champs suit les même règles que le précèdent.
Par exemple:
old/%type%-%id%-%version%
Chemin vers le dépôt
C'est le chemin complet (depuis la racine) du dossier ou seront créées les sauvegardes. Il est préfèrable et même utile d'utiliser le dossier public.
Ce dossier doit être accessible en écriture.
Fichiers suplémentaires à exclure du paquetage
Simple liste de nom à exclure des fichiers. Le joker * peut être utilisé dans ces noms. Les noms sont séparées par des point-virgules.
Par exemple:
*.zip;*.tar;*.tar.gz
Par défaut les fichiers suivants sont exclus:
.;..;__MACOSX;.svn;CVS;.DS_Store;Thumbs.db.
